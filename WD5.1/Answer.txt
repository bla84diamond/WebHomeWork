1.	Для чего нужна эта строка: 
<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">

Эта строка представляет собой метатег для определения масштабирования страницы в мобильных браузерах. 

Атрибут "name" определяет тип метатега, который в данном случае является "viewport".

Атрибут "content" содержит значения, которые определяют, как страница будет отображаться на экране мобильного устройства. Он включает в себя:

- "width=device-width" - устанавливает ширину страницы на устройстве равной ширине экрана устройства.
- "initial-scale=1" - устанавливает начальный масштаб страницы равным 1.
- "shrink-to-fit=no" - запрещает масштабирование страницы для соответствия размеру экрана устройства.

Эта строка очень важна для создания адаптивных веб-сайтов, которые должны хорошо отображаться на различных устройствах с разными размерами экранов.

2.	Можно ли работать с библиотеками локально?

Да, можно работать с библиотеками Bootstrap локально. Для этого нужно скачать файлы библиотеки Bootstrap с официального сайта и разместить их на своем компьютере. Затем можно подключить эти файлы к своему проекту через теги <link> и <script> в HTML-файле. Например:


<link rel="stylesheet" href="path/to/bootstrap.css">
<script src="path/to/bootstrap.js"></script>


Важно убедиться, что пути к файлам указаны правильно и соответствуют фактическому расположению файлов на компьютере. Также следует учесть, что при использовании локальных файлов библиотеки Bootstrap может отсутствовать возможность автоматического обновления при выходе новых версий, поэтому необходимо следить за обновлениями вручную.
